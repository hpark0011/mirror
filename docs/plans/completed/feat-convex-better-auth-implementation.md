# feat: Convex + Better Auth Implementation for Mirror App

**Date:** 2026-01-28
**Branch:** `mirror/012726-feat-auth`
**Target App:** Mirror (`apps/mirror`)

---

## Overview

Implement authentication infrastructure for the Mirror app using **Convex** as the backend and **Better Auth** for authentication. This establishes the foundation for the feel-good monorepo's authentication architecture, with email/password, magic link, and Google OAuth support.

## Problem Statement

We're standardizing on Convex for several reasons:

- **Affordability** - Better pricing for small teams
- **Simpler DX** - No SQL management, TypeScript-native
- **Better Integration** - First-class Next.js and TypeScript support

Mirror is a fresh Next.js 16.1.4 app that needs authentication from scratch. We'll build with Convex + Better Auth from the start.

### Key Technical Decisions

| Decision              | Choice                              | Rationale                                 |
| --------------------- | ----------------------------------- | ----------------------------------------- |
| **Auth Library**      | Better Auth 1.4.9                   | Required for Convex compatibility         |
| **Backend**           | Convex                              | Simpler DX, affordable, TypeScript-native |
| **Email Provider**    | Resend                              | Simple API, good deliverability           |
| **Session Strategy**  | JWT in httpOnly cookies             | Secure, works with middleware             |
| **Package Structure** | Separate convex + features packages | Separation of concerns, reusability       |

### Constraints

1. **Better Auth version pinned to 1.4.9** - Required for `@convex-dev/better-auth` compatibility
2. **Sign in/up must be client-side** - Convex functions use websockets; auth uses HTTP
3. **Convex requires Node.js runtime** - Middleware uses Edge by default; auth verification needs Node

`convex/auth.ts`:

```typescript
import { createClient, type GenericCtx } from "@convex-dev/better-auth";
import { convex } from "@convex-dev/better-auth/plugins";
import { magicLink } from "better-auth/plugins";
import { components, api } from "./_generated/api";
import { type DataModel } from "./_generated/dataModel";
import { betterAuth } from "better-auth";

const siteUrl = process.env.SITE_URL!;

export const authComponent = createClient<DataModel>(components.betterAuth);

export const createAuth = (ctx: GenericCtx<DataModel>) => {
  return betterAuth({
    baseURL: siteUrl,
    database: authComponent.adapter(ctx),

    emailAndPassword: {
      enabled: true,
      minPasswordLength: 8,
    },

    socialProviders: {
      google: {
        clientId: process.env.GOOGLE_CLIENT_ID!,
        clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
      },
    },

    plugins: [
      convex({ jwtExpirationSeconds: 900 }),
      magicLink({
        sendMagicLink: async ({ email, url }) => {
          await ctx.runAction(api.email.sendMagicLink, {
            to: email,
            link: url,
          });
        },
        expiresIn: 900, // 15 minutes
      }),
    ],
  });
};
```

#### 3.4 Auth Hooks

`auth/hooks/use-session.ts`:

```typescript
"use client";

import { useEffect, useState } from "react";
import type { AuthClient } from "../client";

export function createUseSession(authClient: AuthClient) {
  return function useSession() {
    const [session, setSession] = useState<
      typeof authClient.$Infer.Session | null
    >(null);
    const [isLoading, setIsLoading] = useState(true);

    useEffect(() => {
      authClient.getSession().then(({ data }) => {
        setSession(data);
        setIsLoading(false);
      });

      return authClient.onSessionChange((newSession) => {
        setSession(newSession);
      });
    }, []);

    return {
      session,
      user: session?.user ?? null,
      isLoading,
      isAuthenticated: !!session,
    };
  };
}
```

#### 4.2 Middleware

**middleware.ts:**

```typescript
import { NextRequest, NextResponse } from "next/server";
import { getSessionCookie } from "better-auth/cookies";

const PUBLIC_ROUTES = ["/", "/sign-in", "/sign-up"];
const AUTH_ROUTES = ["/sign-in", "/sign-up"];

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;
  const sessionCookie = getSessionCookie(request);
  const isAuthenticated = !!sessionCookie;

  // Redirect authenticated users away from auth pages
  if (isAuthenticated && AUTH_ROUTES.includes(pathname)) {
    return NextResponse.redirect(new URL("/dashboard", request.url));
  }

  // Redirect unauthenticated users from protected routes
  if (!isAuthenticated && !PUBLIC_ROUTES.includes(pathname)) {
    const signInUrl = new URL("/sign-in", request.url);
    signInUrl.searchParams.set("next", pathname);
    return NextResponse.redirect(signInUrl);
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|.*\\..*).*)"],
};
```

## Acceptance Criteria

### Functional Requirements

- [ ] **Email/Password Auth**

  - [ ] User can sign up with email and password (min 8 chars)
  - [ ] User can sign in with existing credentials
  - [ ] Invalid credentials show appropriate error message
  - [ ] Duplicate email on sign-up shows error

- [ ] **Magic Link Auth**

  - [ ] User can request magic link by entering email
  - [ ] Email is sent via Resend within seconds
  - [ ] Clicking link authenticates user and redirects to dashboard
  - [ ] Expired links (>15 min) show error message

- [ ] **Google OAuth**

  - [ ] User can click "Sign in with Google"
  - [ ] OAuth flow completes and redirects to dashboard
  - [ ] New users are created in database
  - [ ] Existing users (by email) are logged in

- [ ] **Session Management**

  - [ ] Session persists across page refreshes
  - [ ] User can sign out
  - [ ] Expired sessions redirect to sign-in

- [ ] **Route Protection**
  - [ ] Unauthenticated users cannot access /dashboard
  - [ ] Authenticated users are redirected from /sign-in to /dashboard
  - [ ] Deep links preserved through auth flow via `?next=` param

### Non-Functional Requirements

- [ ] **Performance**: Auth forms respond within 200ms to user input
- [ ] **Security**: Passwords never logged; tokens in httpOnly cookies
- [ ] **Accessibility**: Forms navigable by keyboard; ARIA labels present
- [ ] **Mobile**: Auth pages responsive on mobile devices

### Quality Gates

- [ ] All auth flows manually tested
- [ ] No TypeScript errors
- [ ] ESLint passes
- [ ] Environment variables documented
